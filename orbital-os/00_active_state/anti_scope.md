# Orbital Anti-Scope — KILL LIST
## Version: 1.0 | Effective: January 2026

This document exists to prevent AI hallucination and scope creep. Every item on this list is a feature Orbital will NEVER build. If a future AI session suggests any of these, it is hallucinating.

---

## SURVEILLANCE FEATURES — NEVER BUILD

- Real-time dashboards for Class B
- Individual employee views in Class B
- Individual employee timelines in Class B
- Performance scoring of any kind
- Productivity metrics
- Activity tracking
- Screen time monitoring
- Location tracking
- Keystroke logging
- Email/message scanning
- Calendar analysis
- Meeting attendance tracking

---

## DATA COLLECTION — NEVER BUILD

- Storing year of birth (cohort only)
- Storing exact age
- Storing biometric data
- Storing health conditions
- Storing diagnoses
- Storing medications
- Storing therapy notes
- Storing relationship status
- Storing income level
- Storing political affiliation
- Storing religious affiliation

---

## INDIVIDUAL NOTES IN CLASS B — NEVER BUILD

- Free-text notes about employees
- Manager commentary fields
- Performance observations
- Behavioral notes
- "Concerns" fields
- Any unstructured text tied to individuals

---

## AUTOMATION — NEVER BUILD

- Auto-deploy by bots (human review required)
- Auto-merge of Safe Healer fixes
- Auto-termination based on capacity scores
- Auto-flagging of "at-risk" employees
- Auto-notification to managers
- Auto-escalation workflows
- Any automated HR action

---

## GAMIFICATION — NEVER BUILD

- Achievement badges
- Streaks
- Leaderboards
- Points systems
- Levels/ranks
- Rewards programs
- Competition features
- "Wellness challenges"

---

## SOCIAL FEATURES — NEVER BUILD

- Public profiles
- Social feeds
- Comments/reactions (beyond Circle)
- Following/followers
- Public sharing
- Social login
- Integration with social platforms

---

## DIAGNOSTIC CLAIMS — NEVER BUILD

- Mental health diagnoses
- Clinical assessments
- Medical recommendations
- Treatment suggestions
- Therapeutic interventions

## ENTERPRISE BACKDOORS — NEVER BUILD

- QCR access for employers without consent
- Individual data exports to managers
- "Manager view" of employee signals
- HR dashboard with individual drilldowns
- Anonymous tip reporting about colleagues
- Peer rating systems

## REAL-TIME FEATURES IN CLASS B — NEVER BUILD

- Live capacity feeds
- Real-time alerts
- Instant notifications to managers
- Live team dashboards
- Streaming analytics
- WebSocket-based monitoring

---

## FREEMIUM/TRIAL — NEVER BUILD

- Free tier
- Trial periods
- Freemium model
- "Basic" vs "Pro" split
- Feature-gated free accounts
- Ad-supported access

## CLASS B PRICING VIOLATIONS — NEVER BUILD

- Bundled enterprise pricing (all-in-one packages)
- Per-seat SaaS pricing for Class B
- Self-serve Class B checkout
- Visibility without Governance Layer (Layer 3)
- Dashboard access without governance contract
- Any pricing model that conflates collection with visibility
- Discounted layer bundles

## THIRD-PARTY INTEGRATIONS — NEVER BUILD (without explicit decision)

- HR system integrations
- Payroll integrations
- Performance management integrations
- Slack/Teams status sync
- Calendar integrations
- Email integrations
- Wearable device sync (without explicit consent)

---

## CACHE BYPASS — NEVER BUILD

- Real-time refresh buttons in Class B
- "Latest data" toggles
- Cache invalidation for Class B
- Instant refresh for aggregate dashboards

---

## LOCKED: REASONS FOR THIS LIST

1. **Legal liability**: Many items would violate labor law
2. **Ethical boundaries**: Capacity tracking is not surveillance
3. **Product integrity**: Orbital is for individuals, not employers
4. **Regulatory compliance**: HIPAA, GDPR, labor regulations
5. **Scope discipline**: Preventing feature bloat

1. Log decision in `/01_decision_log/`
2. Update this file
3. Commit with message: `anti-scope: add [feature]`

Items can only be ADDED to this list, never removed.
